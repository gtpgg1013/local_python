- #### 프로젝트의 이름은 서울 테마카페 맵입니다. 

  - #### MVC 패턴으로 구현

    - #### Model : DB / View : JSP / C : Servlet으로 구현

    - 코드 재사용성 / 유지보수가 쉬움

    - 뷰와 컨트롤러를 따로 독립적으로 관리해서 의존성을 낮게한다

      - 이를 통해서 로직에 영향 주지 않고 화면 표시를 수정할 수 있고, 반대도 된다
      - 컨트롤러 수정시에 뷰를 안건드려도 되고, 반대도 똑같

  - 이것은 사용자가 원하는 테마카페를 찾게 해주는 웹 어플리케이션입니다.

  - 저는 이것을 개발하기 위해 크게 과정을 4스텝으로 나누어 진행했습니다.

  - #### 첫쨰, R 관련 작업들을 먼저 했습니다. 

    - 이를 이용해 데이터를 수집하고, 데이터 정제를 했습니다. Rselenium과 Rvest 패키지를 이용해서 네이버 지도에서 크롤링을 수행하여 csv파일로 저장하였습니다.
    - R leaflet 패키지를 이용하여 google geocoding api 좌표값들로 지도 위에 카페들을 표현하였으며, 추후에 Rjava로 연동하여 웹 위에 표현하였습니다.

  - #### 둘째, Oracle DB를 구축했습니다

    -  카페명, 주소, url 등 기본정보를 저장하는 스키마를 구성하였습니다. 또한 주제 및 방문 키워드, 이용자 리뷰수치(네이버) 등의 스키마를 작성하였습니다.

  - #### 셋째, 자바 VO와 DAO 클래스를 작성하였습니다.

    - VO 클래스는 데이터를 전달하는 역할
    - DAO 클래스는 JDBC 드라이버를 이용하여 oracle과 java와 커넥션을 만들어서 DB내의 내용을 읽어올 수 있게 함 (VO 객체의 형태로)

  - #### 넷째, 서블릿/jsp 파일을 작성하였습니다.

    - 서블릿은 컨트롤러 역할, jsp는 보여주는 역할
    - 서블릿은 자바 파일 안에 html을 작성, - 보이지 않는 내부적 동적인 부분
      - 여기서 데이터를 요청해서 그걸 묶어서 jsp로 던져줌
    - jsp는 html안에 자바 파일을 작성 - 보이는 부분

- 프로젝트 개요

  - '서울 테마카페 맵' 이라는 자바 Servlet 기반 웹 어플리케이션을 만들었습니다.
  - 지역/테마별 카페 추천서비스
  - 이때 배웠던 기술이 데이터 수집과 정제 / 자바를 이용한 웹 어플리케이션 제작

- 프로젝트 과정

  - 1. R 관련 작업을 먼저 했습니다. 

    - R selenium과 Rvest 패키지를 이용해서 크롤링 프로그램 제작하였습니다.

    - 이를 이용해 네이버 지도에 있는 서울 테마카페의 정보들을 크롤링 했습니다.
    - 또한 인스타그램 크롤러를 제작해 특정 까페의 해시태그 정보를 수집해 저희의 방식으로 가공하여 각 점수를 매겨 서비스를 제공했습니다.
    - R leaflet 패키지를 이용하여 지도를 구현하였고 각 카페의 마커 표시는 google geocoding api를 이용해 얻은 좌표값 csv 파일을 이용해 지도 위에 표현하였습니다.
    - 또한 이것을 Rjava를 이용해 java와 연동하였습니다.

  - 2. DB를 구축했습니다. (Oracle DB)

    - 카페명, url, 주소, 전화번호, 인스타그램 등의 기본정보를 저장하는 스키마를 구성하였습니다.
    - 그 다음 카페 분위기, 주제 및 방문목적 키워드, 이용자 리뷰수치 점수를 부여하는 스키마를 구성하였습니다.

  - 3. java DAO class와 VO class를 작성하였습니다.

    - VO(value object)
      - DB(oracle) 과 DAO 클래스를 연결하는 데이터 집합
      - 아까 말했던 DB의 내용들을 전달
    - DAO(Data Access Object)
      - JDBC 드라이버를 이용해서 VO 객체 또는 VO 객체를 담은 리스트들을 전달해줍니다.

       : 상위 클래스가 가지고 있는 메소드를 하위 클래스가 재정의 해서 사용하는 것.
